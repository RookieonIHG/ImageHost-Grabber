ImageHost Grabber

Info

Vielen Foren und Blogs fehlt es an Speicherkapazität zum Speichern von Bildern auf ihren eigenen Servern. Stattdessen müssen Teilnehmer zur Freigabe von Bildern auf Drittanbieter-Hosts zurückgreifen. Es gibt eine Menge kostenloser Bilder-Hosts wie Imagevenue, Imagebam oder Imageshack, mit denen Bilder der Allgemeinheit relativ problemlos zur Verfügung gestellt werden können. Allerdings stellen diese Bilder-Hosts keine direkten Links auf gespeicherte Bilder zur Verfügung. Stattdessen verweisen sie mit Links auf HTML-Seiten, in welche die Bilder integriert werden. Wenn Sie nun eine komplette Bildergalerie dieser Bilder-Hosts speichern möchten, reichen einfache Werkzeuge wie „Wget“ oder „DownThemAll“ nicht aus – jedenfalls nicht ohne beträchtlichen Scripting-Aufwand.

ImageHost Grabber (IHG) ist eine leistungsstarke Anwendung, die das Herunterladen solcher Galerien zum Kinderspiel macht. IHG erledigt dabei alles für Sie: Neuen Link öffnen, rechtsklicken, Speichern unter... Und das für jedes einzelne Bild der Galerie. Aber das ist noch längst nicht alles.

IHG erstellt automatisch Ordner für Sie und benennt Dateien und Ordner so um, dass heruntergeladene Bilder chronologisch geordnet und in der Reihenfolge angezeigt werden, wie sie auf der Originalseite erschienen. Wir haben uns jahrelang mit Problemen beim Download von Bildern beschäftigt und all unsere Erkenntnisse in IHG implementiert, damit alle zur Verfügung stehenden Bilder auch so heruntergeladen werden, wie es sein soll. Und zusätzlich kann IHG auch ganze Sitzungen speichern und sämtliche Bilder auch zu einem späteren Zeitpunkt herunterladen. In IHG finden Sie auch eine spezielle Funktionen für Foren, mit der Sie auch einen gesamten Bilder-Thread herunterladen können. Und zum guten Schluss lädt IHG gleich mehrere Dateien auf einmal herunter, damit alles möglichst flott vonstatten geht.

In IHG finden Sie einen Hostdateieditor, mit dem Sie auch neue, bisher unbekannte Bilder-Hosts einbinden können. Details dazu finden Sie in der „Hostdatei-Anleitung“ auf der IHG-Webseite.

Bitte beachten Sie, dass dieses Programm keine Anhänge oder Links zu Bildern herunterlädt. Es dient lediglich zum Herunterladen von Bildern, die auf kostenlosen Bilder-Hosts abgelegt wurden.

Im Laufe dieser Hilfedatei benutzen wir den Ausdruck„Bilder“, wenn auf Bilder-Hosts abgelegte Bilder gemeint sind.

Hauptfunktionen

Alle Bilder dieser Seite herunterladen

Mit dieser Funktion laden Sie sämtliche Bilder der derzeit angezeigten Seite herunter.

so funktioniert’s:

Öffnen Sie die Seite, von der Sie Bilder herunterladen möchten. Klicken Sie auf „Alle Bilder dieser Seite herunterladen“, wählen Sie anschließend den Ordner, in dem die Bilder gespeichert werden sollen.

 

Aktuellen Thread saugen

Diese Funktion lädt Bilder von gleich mehreren Seiten eines Threads oder einer Galerie herunter – wir nennen das „saugen“. Die Forensoftware oder Galerie muss dabei von IHG unterstützt werden. Folgende Forensoftware und Galerien werden unterstützt: vBulletin, Studio31, Imagebeaver, Facebook, Campusbug, Myspace, UBB Threads, PornBB, Simple Machines Forum, ProBoards, Coppermine, e-hentai, Gelbooru, and Twitpic.

so funktioniert’s:

Rufen Sie eine beliebige Seite des Threads auf, den Sie saugen möchten. Klicken Sie auf den Menüpunkt „Aktuellen Thread saugen“, geben Sie die Nummer der ersten und letzten zu saugende Seite ein, wählen Sie dann den Download-Basisordner.

Bei Bildergalerien öffnen Sie das Album, dessen Bilder Sie speichern möchten. Dann benutzen Sie den Threadsauger wie in einem normalen Forum. Wählen Sie die gewünschten Seiten des Albums.

 

Hostdateieditor

Mit dieser Funktion können Sie die IHG-Hostdatei bearbeiten. Sie können die Hostdatei aktualisieren, eine neue Hostdatei auswählen oder die bereits vorhandenen Einträge in der Hostdatei modifizieren.

so funktioniert’s:

Detaillierte Hinweise finden Sie im Abschnitt „Hostdateieditor“.

 

Gespeicherte Sitzung wiederherstellen

Diese Funktion dient zum Fortsetzen einer Sitzung, die Sie zuvor im Downloadfortschritt-Fenster gespeichert haben.

so funktioniert’s:

Detaillierte Hinweise dazu finden Sie im Abschnitt „Downloadfortschritt-Fenster“.

 

SymbolleistenSCHALTFLÄCHE

Sie können eine spezielle ImageHost Grabber-Schaltfläche in die Hauptsymbolleiste aufnehmen und so Bilder mit nur einem Mausklick bequem herunterladen. Ein Klick auf diese Schaltfläche wirkt so, als würden Sie „Alle Bilder dieser Seite herunterladen“ auswählen.

so funktioniert’s:

Öffnen Sie „Ansicht “/„Symbolleisten“ in der Firefox-Menüleiste, klicken Sie auf „Anpassen...“. Blättern Sie bis zur ImageHost Grabber-Schaltfläche herunter, ziehen Sie diese Schaltfläche anschließend an die gewünschte Stelle in der Firefox-Symbolleiste. Weitere Hinweise zu diesem Thema finden Sie in der Firefox-Hilfedatei; alternativ können Sie mir auch eine eMail (in Englisch) schicken.


 

Optionen

Datei-Speicheroptionen

Dateinamen zur Beibehaltung der Seiten-Bildreihenfolge mit einem Präfix versehen

Diese Option stellt den Dateinamen eine bestimmte Zeichenfolge voran, die dafür sorgt, dass die Dateien so sortiert werden, dass sie wie auf der Seite angezeigt gelistet werden. Der erste Teil dieses Präfixes besteht aus acht alphanumerischen Zeichen, die zur Unterscheidung der Sitzung dienen. Jede Sitzung wird mit einem anderen Präfix versehen – auch dann, wenn mehrere Sitzungen gleichzeitig abgearbeitet werden. Auch im Threadsauger-Modus erhält jede Seite einen eindeutigen Präfix. Der zweite Teil besteht aus einer Zahl, die der Position des Bildes auf der Seite entspricht. Diese Zahl wird mit führenden Nullen aufgefüllt, damit auch weniger logisch arbeitende Sortieralgorithmen (wie in der Zeit vor Windows XP und bei Unix-Betriebssystemen) die Dateinamen richtig sortiert anzeigen.

 

Was soll geschehen, wenn eine Datei bereits am Zielort vorhanden ist?

Es gibt drei Optionen: Datei umbenennen, Skip Datei und Datei überschreiben. Beim Umbenennen der Datei (en) automatisch, fügt die Erweiterung eine Inkrementierung Zahl an das Ende des Dateinamens.

 

Imagevenue- und Fapomatic-Dateien automatisch umbenennen

Diese Funktion war ursprünglich ein Teil der „Auto-Umbenennen “-Funktion in Vorgängerversionen. Dabei werden die führenden Ziffern, die von diesen beiden Servern hinzugefügt werden, vom Dateinamen entfernt. Weitere Server können in künftigen Versionen ergänzt werden.

 

Separate Unterordner mit Dokumenttitel im Zielordner erstellen

Diese Option liest den Titel des Dokumentes aus und nutzt diesen als Bezeichnung eines neuen Unterordners, in dem die Dateien gespeichert werden. Der Titel eines Dokumentes ist der Name der Seite, der oben im Firefox-Fenster angezeigt wird. In den meisten Fällen ergibt sich dadurch eine gute Möglichkeit, unterschiedliche Threads auseinanderzuhalten, ohne die einzelnen Namen mühsam eintippen zu müssen. Allerdings können dabei hin und wieder ziemlich lange Ordnernamen entstehen, manchmal können die Ergebnisse auch nicht eindeutig genug ausfallen.

 

Ordner zur eindeutigen und chronologischen Identifizierung mit einem Präfix versehen

Diese Option setzt einen Zeitstempel vor den Ordnernamen, sofern Sie die vorherige Option nutzen. Dadurch können Ordner chronologisch – also in der Reihenfolge, in der sie heruntergeladen werden – angezeigt werden, ohne sie vorher von Hand sortieren zu müssen. Eine einfache Sortierung nach dem Namen (Standard bei den meisten Anwendungen) tut es.

 

Beim Einsatz des Threadsaugers separate Ordner pro Seite erzeugen

Wenn Sie den Threadsauger nutzen, erzeugt diese Option einzelne Ordner für jede Seite eines Threads oder einer Galerie. Die Ordnernamen entsprechen dabei den Seitennummern. Wenn diese Option nicht aktiviert ist, werden sämtliche Bilder im Basisordner (oder im automatisch erstellten Ordner) gespeichert.

 

Allgemein

Downloadordner-Auswahlfenster anzeigen

Diese Option unterdrückt die Anzeige des Dialogfensters, das Sie fragt, wo die Bilder gespeichert werden sollen. Wenn diese Option nicht markiert ist, werden die Bilder im zuletzt benutzten Ordner gespeichert.

 

Image Grabber-Menü im Extras-Menü anzeigen

Mit dieser Option können Sie auswählen, ob das ImageHost Grabber-Menü im Extras-Menü oder in der Hauptmenüleiste angezeigt wird.

 

Identische Links (auch seitenübergreifend) entfernen

Diese Option ermöglicht Ihnen das Entfernen von identischen (doppelten) Links auf einer Seite. Zu doppelten Links kommt es gewöhnlich, wenn Nutzer Einträge kommentieren und in geistiger Umnachtung auch die Bilder mit zitieren. Wenn Sie den Threadsauger benutzen, sucht ImageHost Grabber sämtliche beteiligten Seiten nach identischen Links ab.

 

CLOSE WINDOW DOWNLOAD WENN ALLE DOWNLOADS fertig sind

Diese Option wird das Download-Fenster schließen, nachdem alle Downloads abgeschlossen haben (nach 5 Sekunden Verzögerung). Wenn es Elemente, die nicht zu Ende war erfolgreich sind, wird das Fenster nicht schließen.

 

Eingebettete Bilder

DOWNLOAD Eingebettete Bilder

Diese Option ermöglicht es Ihnen, Bilder, die download "embedded" auf der Seite. Bei der Verwendung von "Get alle Fotos auf der Seite", wird nur ImageHost Grabber Download der eingebetteten Bilder, deren Abmessungen größer als oder gleich dem angegebenen Abmessungen her durch die "Breite set" und "Höhe" Schaltkästen. Bei Verwendung der Thread Sucker, wird ImageHost Grabber Download nur das eingebettete Bilder, deren Dateigrößen sind größer oder gleich der angegebenen Größe der Datei in der "Dateigröße" Kontrolle-Kästchen.

 

Host-Datei

Enable automatic update

Diese Option wird in der Online-Host-Datei zu verbinden, um neue Hosts zu überprüfen, jedes Mal ImageHost Grabber wird ausgeführt. Sie können wählen, Bestätigungsdialog unterdrücken, als auch die Änderung der Merge-Verhalten.

 

HostOptionen

Maximalzahl gleichzeitiger HTTS-Anfragen

Dies ist die Anzahl von Verbindungen, die gleichzeitig zu einem Bilderhost hergestellt werden. In der Vergangenheit konnte die Anzahl von Downloads von einem Bilderhost nicht gesteuert werden – lediglich die Anzahl von HTTP-Anfragen ließ sich in den Griff bekommen. Heute beschränkt diese Option sowohl HTTP-Anfragen als auch Downloads. Eine Anfrage wird an die Seite gesendet, das Bild gefunden und anschließend herunterladen. Wenn Sie diesen Wert auf 1 setzen, wird immer nur ein einziges Bild heruntergeladen – praktisch, wenn Sie eine langsame Modemverbindung mit geringer Bandbreite nutzen. Mit einer höheren Zahl lässt sich die Downloadgeschwindigkeit oft deutlich erhöhen; allerdings können zu hohe Werte – wie 10 und mehr – zu Verbindungsabbrüchen und übersprungenen Downloads führen. Obwohl bei langsamen Verbindungen automatisch Neuversuche gestartet werden, kann die Downloadgeschwindigkeit kräftig abnehmen, wenn immer wieder neu versucht werden muss. Unsere Erfahrung zeigt, dass der Wert 5 bei den meisten Internetverbindungen prächtig funktioniert. Die Anzahl gleichzeitiger Verbindungen kann jedoch auch vom Bilderhost selbst (nicht selten abhängig von der Tageszeit) eingeschränkt werden.

 

Maximale Wiederholungen pro Link

Durch diese Option können Sie festlegen, wie oft der Download einzelner Links probiert werden soll, bevor die Software aufgibt und zum nächsten Eintrag wechselt. Unter folgenden Umständen kann es zu einem Neuversuch kommen:

  • Die Anfrage war unvollständig (Firefox bemerkte davon jedoch nichts)
  • Die Anfrage braucht zu lange (vermutlich gab der Server bereits auf, Firefox jedoch nicht)
  • Die Anfrage wurde vom Server oder von Firefox (warum auch immer) abgebrochen
  • Bei der Anfrage trat ein Fehler auf
  • Der Server sendete eine Datei, die jedoch kein Bild war

Beachten Sie bitte auch, dass es auch aus anderen Gründen zu einem Neuversuch kommen kann: Nicht, weil etwas schief ging, sondern weil eine alternative URL gefunden wurde. In diesen Fällen wird die Anzahl der Wiederholungen nicht beeinflusst.

 

Anfragewartezeit vor Neuversuch (in Sekunden)

Mit dieser Option legen Sie fest, wie lange eine Anfrage probiert wird, bevor es zu einem Neuversuch kommt.

 

Fehlersuche (Debugging)

Konsolenprotokollierung aktivieren

Diese Option schaltet die Konsolenprotokollierung ein. Nutzen Sie diese Option, falls die Software oder eine bestimmte Funktion der Software nicht richtig funktioniert. In diesem Fall werden kritische Fehler in einer Protokolldatei gespeichert, die Sie anschließend an den Entwickler senden können. Dies wirkt sich leicht negativ auf die Systemleistung aus. Diese Option bringt nichts, wenn die Anwendung zwar funktioniert, aber nicht so, wie Sie es sich vorgestellt haben – dabei handelt es sich um einen logischen Fehler, der aber bedeutend schwieriger in den Griff zu bekommen ist. Ein Beispiel für eine Situation, in der diese Option sinnvoll wäre: Sie klicken auf „Alle Bilder dieser Seite herunterladen“, aber es tut sich rein gar nichts.

Hinweis: Nach dem Aktivieren dieser Option muss Firefox neu gestartet werden, damit die Änderungen in Kraft treten.

 

Fehlersucheprotokollierung aktivieren

Diese Option schaltet die Fehlerprotokollierung ein. Nutzen Sie diese Option, wenn die Anwendung zwar funktioniert, nicht aber so, wie Sie es sich vorgestellt haben. Ein Beispiel: Sie klicken auf „Alle Bilder dieser Seite herunterladen“, sämtliche Dateien werden auch brav gespeichert – allerdings sind es nicht die Bilder, die Sie speichern wollten. Hierbei handelt es sich um einen logischen Fehler. Solche Fehler sind deutlich schwieriger zu beheben als reine (oben erwähnte) Syntaxfehler. Aufgrund der Schwierigkeiten, die sich bei der Problembehebung ergeben, werden sehr umfangreiche Daten bei der Programmausführung ausgegeben. Dadurch ergeben sich deutliche Einschränkungen der Systemleistung. Gehen Sie also entsprechend behutsam mit dieser Funktion um. Versuchen Sie es am besten mit Seiten, die nur wenige Bilder (20 oder weniger) enthalten. Und wenn Sie fertig sind, kopieren Sie das Protokoll in einen lokalen Ordner und löschen das Protokoll anschließend wieder. Bei einer einzigen Programmausführung kann die Protokolldatei gut und gerne mehrere 100 kB groß werden.

Hinweis: Nach dem Aktivieren dieser Option muss Firefox neu gestartet werden, damit die Änderungen in Kraft treten.

 

Protokoll löschen

Mit dieser Funktion löschen Sie das Fehlersucheprotokoll. Dies sollten Sie bereits nach einem einzigen Durchlauf erledigen. Die Protokolldatei wächst sonst im Handumdrehen ins Unermessliche an. Allerdings sollten Sie das Protokoll natürlich erst löschen, nachdem Sie die Protokolldatei in einen lokalen Ordner kopiert haben.

 

Protokoll in lokalen Ordner kopieren

Mit dieser Funktion kopieren Sie die Protokolldatei in einen beliebigen Ordner Ihrer Festplatte oder eines anderen Speichergerätes. Der Name der Datei ist fest vorgegeben: „log.txt“ Nach dem Kopieren können Sie die Datei lesen, sich an den Hut stecken oder – was deutlich sinnvoller ist – an den Entwickler senden.


 

Downloadfortschritt-Fenster

Im Downloadfortschritt-Fenster finden Sie Funktionen, die nur hier zur Verfügung stehen. Allgemein ausgedrückt können Sie weitere Jobs hinzufügen, eine Datei nach dem Download anschauen, Downloads neu starten, den Download noch einmal anstoßen, und so weiter.

 

Allgemeine Funktionen
  • Durch Doppelklick auf den jeweiligen Eintrag öffnen Sie eine Datei nach dem Download.
  • Sie markieren mehrere aufeinander folgende Einträge, indem Sie das erste Element der Reihe anklicken, die Umschalttaste gedrückt halten und anschließend auf das letzte Element der Reihe klicken.
  • Mehrere Einträge, die nicht aufeinander folgen, markieren Sie, indem Sie das erste Element anklicken, die Strg-Taste gedrückt halten und weitere Elemente nach Belieben anklicken.
  • Durch Rechtsklicken eines Eintrags (oder einer Auswahl mehrerer Einträge) öffnet sich ein Kontextmenü, mit dem Sie weitere Aktionen auslösen können.

 

Fertige Jobs automatisch entfernen

Diese Funktion löscht abgeschlossene Jobs nach dem Download automatisch aus dem Fenster.

 

Popup unterdrücken

Diese Option verhindert das Öffnen des Downloadfortschritt-Fensters, wenn Sie weitere Download-Jobs hinzufügen. Damit dies funktioniert, muss das Fenster bereits geöffnet sein.

 

Alle fertigen Jobs entfernen

Bewirkt, dass abgeschlossene Download-Jobs gleich nach dem Download automatisch aus dem Fenster entfernt werden.

 

Alle Jobs stoppen

Mit dieser Funktion stoppen Sie sämtliche laufenden Downloads. Dabei werden sämtliche Downloads abgebrochen, unabhängig davon, zu welcher Sitzung sie gehören.

 

Alle gestoppten Jobs fortsetzen

Diese Funktion ermöglicht den Neustart zuvor gestoppter Sitzungen. Dabei werden sämtliche gestoppten Jobs der Reihe nach neu gestartet, bei Bedarf wiederholt, bis die gesamte Sitzung abgearbeitet ist.

 

Sitzung speichern

Mit dieser Funktion können Sie sämtliche Einträge im Fenster zwischenspeichern und später über die „Gespeicherte Sitzung wiederherstellen “-Funktion im ImageHost Grabber-Hauptmenü wieder fortsetzen. Dies ist gerade bei Verbindungsproblemen und anderen Widrigkeiten sehr praktisch und verhindert, dass Ihre sorgfältig zusammengestellten Downloads verloren gehen.

 

Aktionen mit AUSGEWÄHLTEN ELEMENTEN

Sämtliche Aktionen mit ausgewählten Elementen starten Sie, indem Sie mit der rechten Maustaste auf die gewünschten Elemente klicken und anschließend die gewünschte Aktion aus dem Kontextmenü wählen.

 

Open file

Diese Aktion wird nur das Element öffnen, die den Fokus hat. Zum Beispiel, wenn Sie einen Rechtsklick auf das dritte Element in einer Auswahlliste, nur das dritte Element wird geöffnet. Die "Datei öffnen" Aktion übergibt die Kontrolle über das Betriebssystem, um die Datei im Standard-Programm, das den Dateityp Griffe öffnen.

 

Übergeordneten Ordner öffnen

Diese Aktion öffnet den Ordner enthält für das Element, das den Fokus hat.

 

Aus Liste entfernen

Dieser Aktion entfernt die ausgewählten Elemente dauerhaft aus der Liste.

 

Download/Anfrage abbrechen

Mit dieser Funktion können Sie ein oder mehrere Downloads/Anfragen nach Belieben abbrechen. Dies ist zum Beispiel dann praktisch, wenn Ihnen die Zeit davonläuft und Sie es zu einem späteren Zeitpunkt noch einmal probieren möchten.

 

Download/Anfrage wiederholen

Diese Funktion ermöglicht eine Wiederholung von abgebrochenen, abgeschlossenen oder laufenden Downloads. Die Funktion versucht, beim aktuellen Fortschritt der Downloads wieder anzusetzen.

 

Download neu starten

Diese Funktion startet Downloads komplett neu, statt sie an der unterbrochenen Stelle fortzusetzen. Die Datei wird also von Grund auf neu geladen, bereits heruntergeladene Teile werden dabei ignoriert.

 

Wiederholungszähler auf 0 setzen

Diese Funktion setzt den Wiederholungszähler der ausgewählten Elemente auf 0 zurück. Dies ist beispielsweise dann nützlich, wenn es durch Verbindungsprobleme zu einigen Wiederholungsversuchen kam und Sie es noch einmal von Anfang an probieren möchten.

 

Anfragedetails anzeigen

Diese Funktion öffnet ein neues Fenster, in dem Sie detaillierte Angaben zur Anfrage finden – beispielsweise Speicherort der Datei, URL des Miniaturbildes und den mit der Miniaturbildseite verknüpften HTML-Code.

 

Export list to HTML

Mit dieser Taste können Sie die Download-Liste als HTML-Datei zu speichern. Die HTML-Datei kann dann in das Hauptfenster des Browsers und neu, indem Sie "Holen Sie sich alle Bilder auf der Seite" verarbeitet geöffnet werden.

 

MAX. Anfragen und MAX. Wiederholungen

Diese zwei Steuerelemente ermöglichen es Ihnen, sofort die Kontrolle der maximalen Anzahl von Anfragen und Wiederholungen für den Download-Prozess. Die Änderungen werden sofort wirksam.


 

Hostdateieditor

Dieser Abschnitt deckt lediglich die Grundzüge der Benutzeroberfläche und einfachste Bedienschritte ab. Eine ausführliche Anleitung zum Hinzufügen eigener Hosts finden Sie auf der IHG-Webseite und in folgender Datei:  http://hot-honeys.com/HostFileEditor.pdf

 

Hostbezeichnung

Dies ist eine eindeutige Bezeichnung eines Eintrags in der Hostdatei.

 

URL-Muster

Ein regulärer Ausdruck, der dem URL-Muster des jeweiligen Bilderhosts entspricht.

 

Suchmuster

Dies ist der Teil, der Bilder auf der aufgerufenen Seite findet. Im Abschnitt „Hostdateieditor“ auf der Webseite finden Sie umfassende Erläuterungen zum Suchmuster.

 

Datei aktualisieren

Diese Funktion aktualisiert die Hostdatei mit sämtlichen Änderungen, die Sie an vorhandenen Einträgen ausgeführt haben.

 

Host hinzufügen

Fügt einen neuen Eintrag zur Hostdatei hinzu, bestehend aus den ersten drei eingegebenen Feldern.

 

Host löschen

Löscht den ausgewählten Host aus der Hostdatei.

 

hostf-Server modifizieren

Mit dieser Funktion können Sie weitere Quellen für Online-Hostdateiaktualisierungen hinzufügen. Geben Sie einfach die URL der neuen Hostdatei ein.

 

Hostdatei auswählen

Diese Funktion ermöglicht die Auswahl einer externen Hostdatei; also einer Datei, die sich nicht im IHG-Ordner befindet.

 

Auf Standardhostdatei rücksetzen

Diese Funktion bewirkt, dass wieder die im IHG-Ordner abgelegte Hostdatei verwendet wird.

 

Hostdatei zusammenfassen

Ermöglicht eine Vermischung von Einträgen einer externen Hostdatei (also einer Datei, die nicht in IHG geladen wurde) mit den Einträgen der aktuell verwendeten Hostdatei.

 

Von Online-Hostdatei aktualisieren

Diese Funktion ermöglicht den Abruf aktuellster Einträge aus Online-Hostdateien. Den Speicherort solcher Hostdateien können Sie durch Anklicken der „hostf-Server modifizieren“-Schaltfläche entsprechend anpassen.
