ImageHost Grabber

A propos

De nombreux forums et blogs n'ont pas la capacité de stocker des images sur leurs propres serveurs. Au lieu de cela, ils sont tenus d'utiliser un hôte tiers pour partager des images. Beaucoup d’hébergeurs d’images existent, comme imagevenue, imagebam et imageshack. Toutefois, ces hôtes d'image ne donnent pas de liens directs vers des images stockées. Au lieu de cela, ce qui est donné est des liens vers les pages html qui gardent les images. Si vous souhaitez enregistrer une galerie entière d'images qui sont hébergées sur ces hôtes d'image, des outils simples tels que "wget" et "DownThemAll" ne suffira pas (du moins pas sans quelques scripts).

ImageHostGrabber (IHG) est un utilitaire puissant qui rend la tâche de téléchargement de ces galeries une tâche simple. Vous pouvez penser que c'est IHG qui fait tout le travail du "nouveau lien ouvert, cliquez droit, enregistrer-sous". Mais la fonctionnalité d'IHG ne s'arrête pas là.

IHG peut créer automatiquement des répertoires pour vous, et même des noms de fichiers et de répertoires préfix afin que les images que vous téléchargez puissent être triées par ordre chronologique et être triées par ordre des pages originales. Des années de recherche pour la reprise des échecs de téléchargement ont été mises en œuvre dans IHG pour s'assurer que toutes les images disponibles sont téléchargées correctement. Si ce n'était pas suffisant, IHG peut même enregistrer une session pour essayer à nouveau plus tard. IHG possède également une fonction spéciale pour les forums qui vous permettra de télécharger un sujet entier d'images. Et enfin, mais non le moindre, IHG utilise le téléchargement multiple pour augmenter la vitesse de téléchargement des galeries.

IHG inclut un éditeur de fichier hôte afin que vous puissiez ajouter un hébergeur qui n'est pas déjà inclus. Détails sur cette manipulation peut être trouvée dans le "Manuel du fichier hôte" située sur le site d'IHG.

Veuillez prendre note que ce programme ne peut pas télécharger toutes les pièces jointes, des images ou des liens vers des images. Il ne télécharge que les images hébergées sur les hébergeurs d'image libre.

Pour le reste de ce document d'aide, le mot «images» se référera aux images hébergées sur les hôtes. Le mot «hôte» se référa aux hébergements d’images

 

Les hébergeurs d’image pris en charge

Une liste des hébergeurs d'image pris en charge peut être trouvée sur le site principal: http://imagehost-grabber.eu

 

Caractéristiques principales

Obtenir toutes les photos sur la page

Cette fonction télécharge toutes les images de la page en cours
Comment ça marche:

Aller à la page où vous souhaitez télécharger les images. Cliquez sur «Obtenir toutes les photos sur la page», et choisir le répertoire de téléchargement.

 

Aspirer le sujet en cours

Cette fonctionnalité télécharge des images de plusieurs pages dans un sujet ou d'une galerie, c'est ce que j'appelle «aspiré». Le style de forum ou une galerie doit être soutenu par IHG. Les styles de forum et les galeries suivantes sont supportés: vBulletin, Studio31, Imagebeaver, Facebook, Campusbug, Myspace, Threads UBB, Se connecter pour Simple Machines Forum, ProBoards, Coppermine, e-hentai, Gelbooru et Twitpic.
Comment ça marche:

Prenez n'importe quelle page du sujet que vous voulez «aspirer», cliquez sur «Aspirez le sujet en cours» dans le menu, entrez le numéro de la première page à partir duquel vous souhaitez démarrer, et le dernier numéro de la page où vous souhaitez arrêter, et choisissez le répertoire cible où sera enregistré vos images

Pour les galeries d'image, allez dans l'album où vous souhaitez enregistrer des photos. Ensuite, utilisez l’option «aspiré» comme vous le feriez sur un forum. Choisissez les pages de l'album que vous voulez avoir.

 

Éditeur de fichier d’hôte

Cette fonctionnalité vous permet de modifier le fichier hôte pour IHG. Vous pouvez mettre à jour le fichier hôte, choisissez un nouveau fichier hôte, ou modifier des entrées en cours dans le fichier hôte.
Comment ça marche:

Référez-vous à la section spéciale, «éditeur de fichier d’hôte», pour plus de détails sur la façon dont il fonctionne.

 

Restaurer la session cache

Cette fonctionnalité vous permet de reprendre une session qui a été précédemment enregistrée dans la fenêtre de progression du téléchargement.
Comment ça marche:

Référez-vous à la section spéciale, «fenêtre de progression du téléchargement», pour plus de détails sur la façon dont cela fonctionne.

 

Boutons sur la barre d’outils

Vous pouvez maintenant mettre le bouton ImageHostGrabber sur la barre d'outils pour un accès pratique et facile. Il y a actuellement que l’option «Obtenir toutes les photos sur la page».
Comment ça marche:

Pour mettre le bouton ImageHostGrabber sur la barre d’outils. Allez dans le menu affichage > Barre d’outils > Personnaliser. Prenez l’icône ImageHostGrabber et faites le glisser sur votre barre d’outils. Pour l’enlever, faites la manipulation inverse. Pour plus d’aide sur ce sujet, consultez le fichier d’aide de Firefox, ou écrivez-moi.

Note: Pour ceux qui sont sur Firefox 4 et + faites «Alt» sur votre clavier pour retrouver le menu Firefox.

 

Options

Options pour enregistrer le fichier

Préfixe des noms de fichiers afin qu'il reflète l'ordre des pages

Cette option va ajouter un préfixe au nom du fichier afin que les fichiers puissent être triés dans l'ordre où ils sont affichés sur la page. La première partie du préfixe est de huit caractères de long, aléatoire alphanumérique préfixe qui distingue la session. Chaque session aura un préfixe différent, même lors de l'exécution de plusieurs sessions à la fois. Aussi, lorsqu'il est en mode «aspiré le sujet», chaque page aura un préfixe unique. La deuxième partie est un numéro qui correspond à la position de l'image sur la page. Le nombre est rempli par des zéros afin que les algorithmes de tri non logiques puissent trier les fichiers correctement (c'est-à-dire pré-WinXP et UNIX, aromatisé OS).

 

Que faire si le fichier existe dans le répertoire cible

Il y a trois options: Renommer le fichier, Ignorer le fichier, et écraser le fichier. Lorsque vous renommez le fichier automatiquement, l'extension ajoute un nombre incrémenté à la fin du nom du fichier.

 

Renommer automatiquement les fichiers imagevenue et fapomatic

Cette fonctionnalité était disponible dans les versions précédentes. Cela modifie le nom du fichier à partir du fichier original qui retire les premiers chiffres ajoutés par ces deux serveurs. Les autres serveurs pourraient être mis en œuvre dans de futures versions.

 

Créer un sous-répertoire avec le titre du document à l'intérieur du répertoire cible

Cette option prendra le titre du document et l'utilisera comme nom du nouveau sous-répertoire pour sauver les images. Le titre du document est ce que vous voyez en haut de la fenêtre de Firefox (appelé «barre d'application» dans les fenêtres). La plupart du temps, cela devrait être une manière assez unique pour identifier les sujets sans la peine de taper soi-même. Parfois, cela peut produire des noms de répertoires longs, et parfois ce n'est pas assez unique.

 

 

Identifier les préfixe répertoires unique et chronologiquement

Cette option préfixe est un horodata dans le répertoire lorsque vous utilisez l'option précédente. Cela permet à des répertoires d’être triés par ordre chronologique, sans que vous ayez besoin de les trier vous-même. Le tri simple par nom, qui est par défaut pour la plupart des listes, va fonctionner.

 

Créer des répertoires distincts pour chaque page avec l’option «Aspirez le sujet en cours»

Lors de l'utilisation de l’option «Aspirez le sujet en cours», cette option permettra de créer des répertoires pour chaque page d'un sujet ou une galerie. Les noms des répertoires correspondent aux numéros de page. Si ce n'est pas activé, toutes les images seront sauvegardées dans le répertoire de base (ou répertoire créé automatiquement).

 

Général

Afficher le répertoire de téléchargement

Cette option vous permet de supprimer la boîte de dialogue qui vous demande où enregistrer les images. Si vous décochez cette option, les images seront enregistrées dans le dernier répertoire qui a été utilisé.

 

Afficher le menu ImageHost Grabber dans le menu outils

Cette option vous permettra de choisir d'afficher le menu ImageHostGrabber dans le menu "Outils", ou sur la barre du menu principal.

 

Suppression des liens en doubles, même sur plusieurs pages

Cette option vous permettra de supprimer les liens en double sur la page. Ils sont communément trouvés quand un utilisateur fait un commentaire sur une galerie en faisant des citations des images, produisant ainsi des liens en double. Si vous utilisez «aspirer le sujet courant», ImageHostGrabber se penchera sur toutes les pages que vous téléchargez pour trouver des liens en double.

 

Fermer la fenêtre de téléchargement lorsque tous les téléchargements sont terminés

Cette option va fermer la fenêtre de téléchargement après avoir terminé tous les téléchargements (après un retard de 5 secondes). S’il y a des éléments qui ne se terminent pas correctement, la fenêtre ne fermera pas.

 

Images intégrées

Télécharger les images intégrées

Cette option vous permettra de télécharger des images qui sont «intégrées» sur la page. Lorsque vous utilisez «Obtenir toutes les photos sur la page», ImageHostGrabber ne téléchargera que les images intégrées dont les dimensions sont supérieures ou égales aux dimensions spécifiées définies par les paramètres «largeur» et «Hauteur» dans les options. Lors de l'utilisation de l’option «aspirer le sujet courant», ImageHostGrabber ne téléchargera que les images intégrées dont les dimensions sont supérieures ou égales aux dimensions spécifiées définies par les paramètres «largeur» et «Hauteur» dans les options.

 

Fichier d’hôte

Activer la mise à jour automatique

Cette option va se connecter au fichier hôte en ligne, pour vérifier les nouveaux hôtes, une fois qu’ImageHostGrabber est exécuté. Vous pouvez choisir de supprimer la boîte de dialogue de confirmation.

 

Options de l'hôte

MAX. Connexions simultanées

C'est le nombre de connexions à un hébergeur d'image que l'extension fera en une seule fois. Dans le passé, il n'y avait aucun contrôle sur le nombre de téléchargements à partir d'un hôte d'image, il y avait seulement un contrôle sur le nombre de requêtes HTTP. Maintenant, cette option limite les requêtes HTTP et les téléchargements. Il fait une demande à la page, où se trouve l'image, et télécharge ensuite l'image. Si vous deviez définir cette valeur à 1, alors il aurait téléchargé une image à la fois. Augmenter le nombre peut améliorer les performances du téléchargement, mais un trop grand nombre, comme plus de 10, peuvent causer des délais de connexion et des erreurs de téléchargement. Bien que cette extension comprend une fonction automatique annuler/recommencée en cas de connexion longue, cela peut sérieusement ralentir le processus entier si elle a pour tâche de recommencer un grand nombre de connexions. Selon mon expérience, 5 est le nombre d'or pour la plupart des connexions Internet à haut débit. Ce numéro peut également être limité par l'hôte d'image, et l'heure de la journée.

 

MAX. Nombre de tentatives pour chaque lien

Cette option vous permet de spécifier combien de fois il doit retenter chaque lien avant qu'il abandonne. Les conditions suivantes entraîneront un lien pour faire une nouvelle tentative:

 

• demande incomplète (mais Firefox pense qu'il était complet)

• la demande est trop longue (probablement parce que le serveur a abandonné, mais Firefox n'a pas encore abandonné)

• la demande a été annulée par le serveur ou par Firefox (pour une raison inconnue)

• il y avait une sorte d'erreur dans la requête

• le serveur vous envoie un fichier, mais ce n'est pas une image

 

Notez aussi qu'il y a d'autres cas où la requête est remise à la fonction de redémarrage, mais pas parce que quelque chose tournait mal, mais parce qu’une URL alternative a été trouvée. Ces autres cas n'auront aucune incidence sur le nombre de tentatives.

 

Délai d'attente de la requête avant de réessayer (en secondes)

Cette option permet de spécifier combien de temps l'extension doit essayer chaque demande avant d'envoyer la requête à la fonction de redémarrage.

 

Débogage

Activer la console de journalisation

Cette option active les logs de la console. Utilisez cette option si l'extension ne fonctionne pas, ou une caractéristique de l'extension ne fonctionne pas. Ce relevé signale des erreurs fatales sur la console d'erreur dans un fichier journal que vous pouvez ensuite envoyer au développeur. Cela réduira les performances du système légèrement. Cela ne vous aidera pas si l'extension fonctionne, mais ne fait pas ce à quoi vous vous attendiez (c'est ce qu'on appelle une erreur de logique, et il est plus difficile à déboguer). Un exemple où vous souhaitez l’activer ça serait si vous cliquez sur «Obtenir toutes les photos sur la page» et il ne fait rien.

 

Note: Firefox doit être redémarré après l'activation de cette option.

Activer le journal débogage

Cette option active les journaux de débogage. Utilisez cette option si l'extension fonctionne, mais que quelque chose de différent se produit au lieu de quelque chose qui était attendu. Par exemple, vous cliquez sur «Obtenir toutes les images sur la page" et il télécharge tous les fichiers, mais les images qu'il télécharge ne sont pas celles que vous voulez. C'est ce qu'on appelle une erreur de logique. Cela est beaucoup plus difficile à déboguer qu'une erreur de syntaxe (comme décrit ci-dessus). Parce qu'il est plus difficile à déboguer, un débit très détaillé de l'exécution du programme. En conséquence, les performances du système vont baisser très sensiblement. Vous devez être prudent avec cette fonction. Essayez d'utiliser une page avec une petite quantité d'images (soit 20 ou moins). Et lorsque vous avez terminé, une copie du journal dans un dossier local et effacer le journal. Le fichier journal peut arriver à plusieurs centaines de kilo-octets en un seul passage.

 

Note: Firefox doit être redémarré après l'activation de cette option.

Effacer le journal de débogage

Utilisez cette fonction pour effacer le journal de débogage. C'est une bonne idée après avoir fait un seul passage. Le fichier journal sera assez grand. Faites ce nettoyage après avoir copié le fichier log dans un dossier local.

 

Copier le journal de débogage vers un dossier local

Utilisez cette option pour copier le fichier log dans un dossier situé quelque part sur votre disque dur, sur votre bureau ou sur un périphérique de stockage. Vous n'avez pas le choix dans le nom du fichier, ça sera «log.txt». Une fois qu'il est copié, vous pouvez le lire, ou l'envoyer par courriel au développeur.

 

Fenêtre de la progression du téléchargement

La fenêtre de progression du téléchargement contient des fonctionnalités qui ne sont pas disponibles nulle part ailleurs dans ImageHostGrabber. En général, vous pouvez ajouter plusieurs sessions, vous pouvez prévisualiser un fichier après son téléchargement terminé, vous pouvez redémarrer et réessayer des téléchargements, et ainsi de suite.

 

Caractéristiques générales

 

• Vous pouvez ouvrir le fichier après son téléchargement terminé en double cliquant sur l'élément d'entrée

• Vous pouvez sélectionner plusieurs éléments dans la séquence en cliquant sur le premier élément de la séquence, maintenez la touche Maj et cliquez sur le dernier élément de la séquence

• Vous pouvez sélectionner plusieurs éléments (pas dans la séquence) en cliquant sur le premier point, puis tout en maintenant la touche Ctrl, cliquez sur chaque article suivant pour être sélectionnés

• Vous pouvez effectuer des actions sur un élément (ou la sélection des articles) par un clic droit sur ce point (ou tout autre élément de la sélection)

 

Effacement automatique quand c’est fini

Efface automatiquement les téléchargements à partir de la fenêtre, quand les téléchargements sont finis.

 

Supprimer le popup

Cette option supprime le contenu contextuel de la fenêtre de progression du téléchargement lors de l'ajout de plus de téléchargements sur la fenêtre. La fenêtre doit déjà être ouverte pour que cela fonctionne.

 

Effacer tout quand c'est fini

Permet d'effacer tous les téléchargements finis à partir de la fenêtre.

Arrêter toutes les sessions

Cette fonctionnalité vous permettra d'arrêter tous les téléchargements en cours. Il provoque l’abandon de chaque téléchargement, quelle que soit la session.

 

Relancer toutes les sessions arrêté

Cette fonctionnalité vous permettra de relancer les sessions précédemment arrêtées. Chaque séance débutera par les téléchargements préalablement arrêtés, puis continuera avec le reste de la session.

 

Enregistrer la session cache

Cela vous permet de sauvegarder tous les éléments dans la fenêtre et de mettre en cache, ce qui peut être rappelé plus tard en utilisant la fonction «Restaurer la session cache» dans le menu ImageHostGrabber. Ceci est utile si vous rencontrez des problèmes de connexion ou autre chose et vous ne voulez pas perdre tout le temps que vous mettez dans la construction de la liste de téléchargement.

 

Actions sur les éléments sélectionnés

Toutes les actions sur les éléments sélectionnés sont effectuées par un clic droit sur l'élément (s) et en sélectionnant l'action à effectuer.

 

Ouvrir le fichier

Cette action sera ouverte uniquement sur l'élément qui a été sélectionné. Par exemple, si vous faites un clic droit sur le troisième élément de la liste de sélection, seul le troisième point sera ouvert. Le «Ouvrir le fichier» l’action passe le contrôle sur le système d'exploitation pour ouvrir le fichier dans le programme par défaut qui gère le type de fichier.

 

Ouvrez le dossier contenant

Cette action va ouvrir le dossier contenant l'élément qui a été sélectionné

 

Retirer de la liste

Cela vous permet de supprimer un ou plusieurs produit (s) dans la liste.

 

 

Abandonner le téléchargement / requête

Cette fonctionnalité vous permet d'annuler une ou plusieurs requêtes pour une raison quelconque. ça pourrait être une option que vous considérez si le téléchargement est trop long et que vous voulez réessayer ultérieurement.

 

Réessayer de télécharger / requête

Cette fonctionnalité vous permettra de recommencer un ou plusieurs téléchargements qui ont été abandonnés, s'est terminée avec succès, ou qui est en cours. Il va réessayer d’avancer le progrès actuel du téléchargement (s).

 

Redémarrer le téléchargement

Cette fonction redémarre le téléchargement (s) complètement, quel que soit l'état d'avancement. Il commence avec un nouveau téléchargement, il ne reprend pas les progrès antérieurs.

 

Réinitialiser le nombre de tentatives à zéro

Cette fonctionnalité vous permet de réinitialiser le nombre de tentatives de l'élément sélectionné (s). Il est utile s'il y avait des problèmes de connexion qui a gonflé le nombre de tentatives et que vous voulez essayer à nouveau.

 

Afficher les détails de larequête

Cette fonctionnalité vous permettra de visualiser les détails de la requête, par exemple lorsque le fichier est enregistré, l'URL de la vignette, et le code HTML associé à la page des vignettes.

 

Exporter la liste en HTML

Ce bouton vous permet d'enregistrer la liste de téléchargement dans un fichier html. Le fichier html peut ensuite être ouvert dans la fenêtre principale du navigateur et être retraité en exécutant « Obtenir toutes les photos sur la page».

 

Max. Requêtes et Max. tentatives

Ces deux commandes vous permettent instantanément de contrôler le nombre maximum de requêtes et de tentatives pour le processus de téléchargement. Les changements prennent effet immédiatement.

 

Editeur de fichier hôte

Cette section ne couvre que les bases de l'interface pour les opérations de base. Vous pouvez trouver un tutoriel complet sur l'ajout de vos propres hôtes sur le site Web, ou en téléchargeant le fichier suivant:  http://imagehost-grabber.eu/HostFileEditor.pdf

 

Etiquette d'hôte

Ceci est l'étiquette d'identification unique pour une entrée dans le fichier hôte.

 

Modèle d'URL

Ceci est l'expression régulière qui correspond au motif URL de l’hébergeur d’images.

 

Modèle de recherche

C'est la partie qui trouve l'image sur la page résultante. Veuillez-vous référer à l’«Éditeur de fichiers hôte» sur la page principale de discussion sur le modèle de recherche.

 

Mise à jour des fichiers

Cette mise à jour du fichier hôte avec tous les changements que vous avez apportés à une entrée existante.

 

Ajouter un hôte

Cela ajoute une nouvelle entrée à l'information de fichier hôte donné dans les trois premiers champs.

 

Supprimer l'hôte

Ceci supprime l'entrée dans le fichier hôte.

 

 

Modifier les serveurs hôte

C'est là que vous pouvez ajouter d'autres sources hôte. Entrez simplement l'URL de l'emplacement du nouveau fichier hôte.

 

Choisissez un Fichier hôte

Cela vous permet de choisir un fichier hôte autre que celle résidant dans le dossier d'extension d’ImageHostGrabber.

 

Réinitialiser au fichier d'hôte par défaut

Il utilise le fichier hôte qui réside dans le dossier d'extension d’ImageHostGrabber.

Fusionner le fichier hôte

Cela vous permettra de fusionner les entrées dans un fichier hôte (autre que celui que vous utilisez) avec le fichier hôte que vous utilisez actuellement.

 

Mise à jour du fichier hôte en ligne

Cela vous permet de recevoir des entrées les plus à jour à partir des fichiers d'hôte en ligne. Les emplacements en ligne de fichiers hôte peuvent être modifiés en appuyant sur le bouton «Modifier les serveurs hôtes»
