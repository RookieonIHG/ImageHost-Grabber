ImageHost Grabber

About

Many forums and blogs don’t have the capacity to store images on their own servers. Instead, posters are required to use a third-party host to share images. Many free image hosts exist, such as imagevenue, imagebam, and imageshack to serve the purposes of sharing images. However, these image hosts do not give direct links to stored images. Instead, what’s given are links to html pages that encapsulate the images. If you want to save an entire gallery of images that are hosted on these image hosts, simple tools like «wget» and «downthemall» will not suffice (at least not without some serious scripting).

ImageHost Grabber (IHG) is a powerful utility that makes the task of downloading these galleries a simple task. You can think of it as IHG doing all the work of «open new link, right-click, save-as». But the functionality of IHG doesn’t stop there.

IHG can automatically create the directories for you, and even prefix file names and directories so that images you download can be sorted chronologically and be sorted by the original page order. Years of research of download failure recovery has been implemented into IHG to ensure all available images are downloaded properly. If that wasn’t enough, IHG can even save a session to try again later. IHG also includes a special feature for forums that will allow you to download an entire thread of images. And last, but not least, IHG uses multiple downloads to increase the download rate of the galleries.

IHG includes a host file editor so that you can implement a new image host that is not already included. Details on doing this can be found in the «Host File Manual» located on the IHG website.

Please take note that this program will not download any attachments, images, or links to images. It only downloads images hosted on the free image hosts.

For the remainder of this help document, the word «images» will refer to the images hosted on the hosts.

Main Features

Get all pics on page

This feature downloads all the images from the current page being displayed.

How it works:

Go to the page that you want to download the images from. Click the «Get all pics on page», and pick the download directory.

 

Suck the current thread

This feature downloads images from multiple pages in a thread or gallery; this is what I call «sucking». The forum style or gallery must be supported by IHG. The following forum styles and galleries are supported: vBulletin, Studio31, Imagebeaver, Facebook, Campusbug, Myspace, UBB Threads, PornBB, Simple Machines Forum, ProBoards, Coppermine, e-hentai, Gelbooru, and Twitpic.

How it works:

Go to any page of the thread that you want to «suck», click the «Suck the current thread» menu item, enter the first page number you want to start from, the last page number to end at, and pick the base download directory.

For image galleries, go to the album that you want to save pictures from. Then use the thread sucker like you would on a forum. Choose a range of pages from the album that you want.

 

Host File Editor

This feature allows you to modify the host file for IHG. You can update the host file, pick a new host file, or modify current entries in the host file.

How it works:

Please refer to the special section, «Host File Editor», for details on how it works.

 

Restore cached session

This feature allows you to resume a session that was previously saved in the download progress window.

How it works:

Please refer to the special section, «Download Progress Window», for details on how this works.

 

Toolbar Button

You can now put the ImageHost Grabber toolbar button on the main toolbar for easy convenient access. It currently only does the «Get all pics on page».

How it works:

Open the «Customize» item under «View→Toolbars», find the ImageHost Grabber toolbar button, and drag it to the toolbar where you want it to be located. For further help on this topic, check out Firefox’s help file, or email me.


 

Options

File Saving Options

Prefix filenames to reflect page order

This option will add a prefix to the filename so that the files can be sorted in the same order as they are displayed on the page. The first part of the prefix is an eight character long, random alpha-numeric prefix that distinguishes the session. Each session will have a different prefix, even when running multiple sessions at a time. Also, when in thread sucker mode, each page will get a unique prefix. The second part is a number that corresponds to the picture’s position on the page. The number is padded with leading zeros so that non-logical sort algorithms can sort the files properly (i.e. pre-WinXP and UNIX flavored OS’s).

 

What to do if file exists in target location

There are three options: Rename File, Skip File, and Overwrite File. When renaming the file(s) automatically, the extension adds an incrementing number to the end of the filename.

 

Autorename imagevenue and fapomatic files

This functionality was originality part of the «Autorename» feature from previous versions. This modifies the filename from the file to remove the leading digits added by these two servers. Other servers could be implemented here in future versions.

 

Create a sub-directory with document title inside the target directory

This option will take the document title and use that as the name of the new sub-directory to save the media. The document title is what you see at the top of the Firefox window (called the «Application Bar» in windows). Most of the time, this should be a fairly unique way to identify the threads without the trouble of typing it yourself. Sometimes, this can produce lengthy directory names, and sometimes it is not unique enough.

 

Prefix directories to uniquely and chronologically identify them

This option will prefix a time stamp to the directory when using the previous option. This allows the directories to be sorted chronologically without the special effort of sorting them yourself. The simple sorting by name, which is default for most listings, will work.

 

Create separate directories for each page when using thread sucker

When using the thread sucker, this option will create directories for each page of a thread or gallery. The directory names will correspond to the page numbers. If not enabled, all images will be saved in the base directory (or automatically created directory).

 

General

Show the Download Directory box

This option allows you to suppress the dialog that asks you where to save the images. If you uncheck this option, images will be saved to the last directory that was used.

 

Show the Image Grabber menu in the tools menu

This option will allow you to choose whether to show the ImageHost Grabber menu in the «Tools» menu, or on the main menu bar.

 

Remove duplicate links, even across multiple pages

This option will allow you to remove duplicate links on the page. These are commonly found when a user comments on a gallery and quotes the images, thus producing duplicate links. If you’re using the thread sucker, ImageHost Grabber will look at all the pages you are downloading for duplicate links.

 

Close download window when all downloads are finished

This option will close the download window after all downloads have finished (following a 5 second delay). If there are items that did not finish successfully, the window will not close.

 

Embedded Images

Download embedded images

This option will allow you to download images that are «embedded» on the page. When using «Get all pics on page», ImageHost Grabber will only download the embedded images that whose dimensions are greater than or equal to the specified dimensions set forth by the «Width» and «Height» control boxes. When using the thread sucker, ImageHost Grabber will only download the embedded images whose file sizes are greater than or equal to the specified file size in the «File size» control box.

 

Host File

Enable automatic update

This option will connect to the online host file, to check for new hosts, every time ImageHost Grabber is executed. You can choose to suppress the confirmation dialog, as well as change the merge behavior.

 

Host Options

Max. simultaneous connections

This is the number of connections to an image host that the extension will make at one time. In the past, there was no control over the number of downloads from an image host; there was only control over the number of HTTP requests. Now this option restricts both the HTTP requests and downloads. It makes a request to the page, finds the image, and then downloads the image. If you were to set this value to 1, then it would download one image at a time (perfect for you dial-up users). Increasing the number may improve performance of the download; however, too large of a number, like more than 10, can cause connection timeouts and dropped downloads. Although this extension includes an automatic cancel/retry feature for long connection times, this can seriously slow down the entire process if it has to retry a lot of connections. It has been my experience that 5 is the golden number for most high speed internet connections. The number can also be limited by the image host, and the time of day.

 

Max. number of retries for each link

This option lets you specify how many times it should retry each link before it gives up and moves on. The following conditions will cause a link to be retried:

  • incomplete request (but Firefox thinks it was complete)
  • request is taking too long (probably because the server gave up but Firefox didn’t)
  • the request was cancelled by the server or by Firefox (for some un-godly reason)
  • there was some kind of error in the request
  • the server sent you a file, but it’s not an image

Also note that there are other cases where request is turned over to the retry function, but not because anything went wrong, but because an alternate URL was found. These other cases won’t affect the retry count.

 

Request timeout before retry (in seconds)

This option lets to specify how long the extension should try each request before sending the request to the retry function.

 

Debugging

Enable Console Logging

This option turns on the console logs. Use this if the extension doesn’t work, or a feature of the extension doesn’t work. This will record the fatal errors reported on the error console into a log file that you can then send to the developer. This will decrease system performance slightly. This won’t help you if the extension is working, but isn’t doing what you expected it to do (that’s called a logical error, and is harder to debug). An example where you would want to enable this would be if you click on «Get all pics on page» and it does nothing.

Note: Firefox has to be restarted after enabling this option.

 

Enable Debugging Log

This option turns on the debugging logs. Use this if the extension is working but is doing something different than what is expected. For instance, you click «Get all pics on page» and it downloads all the files, but the images it downloads aren’t the ones you wanted. This is called a logical error. This is much harder to debug than a syntax error (as described above). Because it is harder to debug, a very verbose output of the program execution is made. As a result, system performance will drop very noticeably. You should be careful with using this feature. Try to use a page with a small amount of images (i.e. 20 or less). And when you’re done, copy the log to a local folder and clear the log. The log file can get to several hundred kilobytes in a single run.

Note: Firefox has to be restarted after enabling this option.

 

Clear Debug Log

Use this to clear the debug log. This is a good idea after doing a single run through. The log file will get fairly large real quick. You will want to wait to do this until after you copy the log file to a local folder.

 

Copy Debug Log to Local Folder

Use this to copy the log file to a folder located somewhere on your hard drive/desktop/storage device. You do not have a choice in the name of the file; it will be «log.txt». Once it is copied, you can read it, or stick it in your bum or the better choice, email it to the developer.


 

Download Progress Window

The download progress window contains features that are not available anywhere else in ImageHost Grabber. In general, you can add multiple sessions, you can preview a file after it has finished downloading, you can restart and retry downloads, and so on.

 

General Features

  • You can open the file after it has finished downloading by double clicking on the entry item
  • You can select multiple items in sequence by clicking on the first item in the sequence, hold the shift key, and click the last item in the sequence
  • You can select multiple items (not in sequence) by clicking on the first item, then while holding on the control key, click on each subsequent item to be selected
  • You can perform actions on an item (or selection of items) by right clicking on that item (or any item in the selection)

 

Auto Clear Finished

Automatically clears downloads from the window as they are finished downloading.

 

Suppress Popup

This option suppresses the popup behavior of the download progress window when adding more downloads to the window. The window must already be open for this to work.

 

Clear All Finished

Allows you to clear all the finished downloads from the window.

 

Stop All Sessions

This feature will allow you to stop all downloads in progress. It causes each download to abort, regardless of which session.

 

Revive All Stopped Sessions

This feature will allow you to re-start the previously stopped sessions. Each session will start with the previously stopped downloads, retrying them, and then continue with the rest of the session.

 

Save session to cache

This allows you to save all the items in the window to cache, which can be recalled later using the «Restore cached session» feature in the main ImageHost Grabber menu. This is helpful if you’re having connection problems or whatever and you don’t want to lose all the time you put into building the download list.

 

Actions on selected items

All of the actions on selected items are performed by right-clicking on the item(s) and selecting the action to be performed.

 

Open file

This action will open only the item which has focus. For instance, if you right click on the third item in a selection list, only the third item will be opened. The «Open file» action passes control over to the operating system to open the file in the default program that handles the file type.

 

Open containing folder

This action will open the containing folder for the item that has focus.

 

Remove from list

This allows you to remove one or more item(s) from the list.

 

Abort download/request

This feature allows you to cancel one or more requests for whatever reason. This might be an option you would consider if it is taking too long and you want to retry it at a later time.

 

Retry download/request

This feature will allow you to retry one or more downloads that have been aborted, completed successfully, or that is in progress. It will retry from the current progress of the download(s).

 

Restart Download

This feature restarts the download(s) completely, regardless of the progress. It starts with a fresh download, not resuming from any previous progress.

 

Reset the retry count to zero

This feature allows you to reset the retry count of the selected item(s). It is useful if there were connection problems that inflated the retry count and you want to try again.

 

View Request Details

This feature will allow you to view the details of the request, such as where the file is being saved, the thumbnail’s URL, and the HTML associated with the thumbnail page.

 

Export list to HTML

This button allows you to save the download list as an html file. The html file can then be opened in the main browser window and re-processed by running «Get all pics on page».

 

Max. requests and Max. retries

These two controls allow you to instantaneously control the maximum number of requests and retries for the download process. The changes take effect immediately.


 

Host File Editor

This section will only cover the basics of the interface for basic operations. You can find an in depth tutorial on adding your own hosts on the main web site, or downloading the following file:  ImageHost Grabber Host File Editor Manual

 

Host Label

This is the unique identifying label for an entry in the host file.

 

URL Pattern

This is the regular expression that corresponds to the image hosts URL pattern.

 

Search Pattern

This is the part that finds the image on the resulting page. Please refer to the «Host File Editor» on the main page for in depth discussion about the search pattern.

 

Save Changes

This updates the host file with any changes you may have made to an existing entry.

 

Add Host

This adds a new entry to the host file given information entered into the first three fields.

 

Delete Host

This deletes the entry from the host file.

 

Modify hostf servers

This is where you can add other sources of online host file updates. Simply enter the URL of the new host file location.

 

Pick Host File

This allows you to pick a host file other than the one residing in ImageHost Grabber’s extension folder.

 

Reset to Default Host File

This uses the host file that resides in ImageHost Grabber’s extension folder.

 

Merge Host File

This will allow you to merge the entries in a host file (other than the one you’re using) with the host file that you’re currently using.

 

Update from online Host file

This allows you to receive the most up-to-date entries from the online host files. The online host file locations can be modified by hitting the «Modify hostf servers» button.
